This workbench is built for incident response teams that need faster first-pass reporting. Bring in evidence, force structure around the case, run analysis through OpenRouter, and export selected outputs as `.docx`.

Workflow

Three stages from evidence to deliverable

01

Ingest

Load text evidence, paste analyst observations, and pin the case to an incident type or let the model classify it.

02

Analyze

OpenRouter receives a structured case packet and returns normalized incident sections with explicit confidence and action items.

03

Export

Choose only the documents you need and export a zipped deliverable set of `.docx` files for PIRs, summaries, timelines, and remediation plans.

    Security Notes

    Important operational constraints

    Key handling

    GitHub Pages cannot safely hide a private OpenRouter key. This build uses analyst-supplied runtime credentials and stores them only for the current browser session.

    Evidence format

    This version reads text evidence directly in the browser. Binary forensic artifacts still need preprocessing or a backend parser if you want deep automation later.

    PIR meaning

    This app treats PIR as a Post Incident Report. If your internal PIR format means something else, the prompt schema can be adjusted without changing the export pipeline.
