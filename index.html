Workflow

Three stages from evidence to deliverable

01

Ingest

Load text evidence, paste analyst observations, and pin the case to an incident type or let the model classify it.

02

Analyze

The analysis engine reviews the case packet and returns normalized incident sections, timelines, indicators, and action items.

03

Export

Choose only the documents you need and export a zipped deliverable set of `.docx` files for PIRs, summaries, timelines, and remediation plans.

    Operational Notes

    Implementation notes and handling guidance

    Key handling

    This site uses a user-supplied session key. If the page remains public, keep that key local and avoid committing it to the repository.

    Evidence format

    This version reads text evidence directly in the browser. Binary artifacts still need preprocessing or a separate parsing workflow.

    Report naming

    Within this workspace, PIR refers to a Post Incident Report. The output structure can be adjusted later without changing the export flow.
